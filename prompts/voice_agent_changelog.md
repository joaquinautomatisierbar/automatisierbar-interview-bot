- **v7.1 · 2026-06-06 · Bookable: appointment fields in structured-data extraction** — Lena booked verbally but nothing structured came out for a confirmation/scheduling backend to act on. Extended the assistant's `analysisPlan.structuredDataPlan.schema` (was: connected/interest_level/pain_score/interview_accepted/etc.) with 5 booking fields: `appointment_booked` (bool — only when day+time agreed AND confirmed), `appointment_day` (text as said), `appointment_time` (24h), `contact_channel` (sms/email/phone), `contact_value` (email/number; "inbound_number" if prospect says "use the number you called"). These land in `call.analysis.structuredData` on the existing `end-of-call-report` webhook → `https://oojoaquin.app.n8n.cloud/webhook/voice-call-result`. Handoff for the in-progress confirmation backend: read those fields; resolve `appointment_day`+`appointment_time` to a real datetime using the call's `createdAt` (extractor doesn't know "today"). Caller-ID/Swiss-number + SMS/confirmation sender being handled separately by operator (Swiss number ~2 days out). *Why:* "she needs to be able to book an appointment to talk about it again" — turns the conversation into consumable booking data.
- **v7 · 2026-06-06 · Honest meeting framing + natural scheduling + scale fix** — Nico v6-call feedback. (1) **Meeting = discovery, not demo.** Lena had over-promised that the 30-min call shows "wie alles funktioniert / was sich ändert" + a finished prototype — but that call is the *Workflow Interview* (business-context sales motion step 3 = understand the process), build happens *after* (step 4), demo is a *later* meeting (step 5). Added a "Was im Termin wirklich passiert" block enforcing the honest order: erst VERSTEHEN (im Termin) → dann BAUEN (danach) → dann unverbindlich TESTEN; explicit "nicht im Termin einen fertigen Prototyp / 'wie alles funktioniert' versprechen." (2) **Scheduling less like an interrogation.** Was 4 sequential questions (week → day → vormittags/nachmittags → exact time). Now: week → day → **one concrete proposal** ("[Tag] am Vormittag, so gegen 9 Uhr? Sonst sagen Sie, was besser passt") → read back once. Per Nico: a suggestion the prospect adjusts sounds far more natural than asking everything. (3) **Scale drift fixed** — Lena improvised "Skala 1 bis 10" in the v6 call; hardened to "IMMER 1 bis 5, nie 1 bis 10." *Open infra (not prompt — flagged to operator):* (a) **caller ID is a US Twilio number** `+1 814 914 9096` (only *named* "Automatisierbar +41 44") → Swiss carriers mislabel it ("Corsica") + low pickup; needs a real +41 number (Twilio CH regulatory bundle). (b) **No SMS/booking-confirmation backend exists** — Lena promised an SMS that nothing sends; booking data dies in the transcript. Needs build: extract booking from call → send SMS/email + optional calendar event.
- **v6 · 2026-06-06 · Objection handling: convert the soft "no" (risk-reversal loop)** — Nico v5-call feedback: he gave an unspecific "keine Interesse" (no real reason) and Lena folded on the first no — "many prospects who don't understand AI need a bit of convincing."
